Как работает интернет: базовые сетевые протоколы

HTTPS - протокол прикладного уровня.

Transmission control protocol работает на транспортном уровне

В адресе типа IPv4 не может быть чисел больше 255.

DNS-сервер.

Распределение протоколов в модели TCP/IP:

  • Прикладной уровень: HTTP, RTSP, FTP, DNS.

  • Транспортный уровень: TCP, UDP, SCTP, DCCP.

  • Сетевой (Межсетевой) уровень: IP.

  • Уровень сетевого доступа (Канальный): Ethernet, IEEE 802.11, WLAN, SLIP, Token Ring, ATM и MPLS.

Протокол http передает не зашифрованные данные (а https передает уже зашифрованные).

Https передает зашифрованные данные, поэтому одна из фаз - передача данных, другая должна быть рукопожатием.

TLS определяется как клиентом, так и сервером.

Персонализация сети

Куки хранят ID ceccии и идентификатор.

Куки не делают соединение более надежным, даже наоборот.

Куки генерируются сервером.

Сессионные куки хранятся лишь в течение сессии

Браузер TOR

Необходимо три узла: входной, промежуточный и выходной.

IP-адрес не должен быть известен охранному и промежуточному узлам.

Отправитель генерирует общий секретный ключ с узлами, через которые идет передача, то есть со всеми.

Для получения пакетов не обязательно использовать TOR.

Беспроводные сети Wi-fi

Wi-Fi.

Wi-Fi располагается как канальный уровень ниже интернет-уровня интернет-протокола.

WEP (Wired Equivalent Privacy) – устаревший и небезопасный метод проверки подлинности.

Нужно аутентифицировать устройства, а потом передать зашифрованные данные.

WPA2 Personal для личного использования, а enterprise - для предпиятий.

Пароли

Стойкий пароль должен быть со специальными символами и достаточной длины.

Менеджер паролей - это самый безопасный вариант.

Капча нужна для проверки на то, что за экраном не бот.

Опасно хранить пароли в открытом виде, для этого есть хэши.

Соль не может помочь для улучшения стойкости паролей к атаке перебором, если злоумышленник получил доступ к серверу.
